# Huddledin — CLAUDE.md ## What This Is Care coordination platform for families managing children with multiple therapists/specialists. Single-file PWA (`index.html`, ~18,750 lines) + esbuild module system for new features. Vanilla JS, Supabase JS v2, Vercel, GitHub auto-deploy. **Live:** huddledin.com **Local:** `C:\Users\eytan\OneDrive\Apps\Huddledin` --- ## Golden Rules 1. **Supabase is the only source of truth.** `DB`/`LS` = display cache. Write Supabase first, then update cache. 2. **Always `{ data, error }` from ALL Supabase calls.** Use `try/catch` with `console.error`. Never swallow errors. 3. **Never `.maybeSingle()`** — use `.limit(1)`. 4. **Never `ignoreDuplicates: true`** on profile upserts — silently drops role writes. 5. **DB Proxy pattern** — `DB` is a Proxy over localStorage. `DB.x[0].y = z` does nothing. Always: read full array → mutate → write back: `const arr = DB.x; arr[0].y = z; DB.x = arr;` 6. **`notifications.id` has NO default** — always provide: `'n_op_' + Date.now() + '_' + userId`. 7. **Never DELETE `specialist_requests`** — UPDATE status only. 8. **Never write `household_id` or `role` to specialist profiles.** 9. **`specialist_requests.household_id` and `folder_permissions.specialist_id` are TEXT** — always `String()`. 10. **i18n is mandatory.** All new user-facing strings: `T('key')`, add to both `_STRINGS.en` and `_STRINGS.he`. No exceptions. 11. **Never use "special needs"** — Huddledin is for all children. 12. **Demo mode:** `session.id === 'demo'` — skip all Supabase writes. --- ## Architecture ### State & Render - **`S`** — single UI state object. Mutate → call `re()`. Never mutate DOM directly. - **`re()`** — wipes `#root`, redraws everything. No diffing. - **`_supa`** — Supabase client. Module-level, never recreate. - **`DB`** — Proxy over localStorage. Read cache only. See Golden Rule #5. - **`refreshFromSupabase()`** — pulls all remote data into cache after auth/realtime. - **`_debouncedRefresh()`** — debounced wrapper. **Always use this in realtime callbacks**, never `refreshFromSupabase()` directly. - **`SB`** — async Supabase CRUD helpers per table. ### Navigation (Parent) - `S.navLevel === 0` → Family Home (no child selected) - `S.navLevel === 1` → Child profile. `S.activeChild` = child ID, `S.activeTab` = section - Use `switchTab(id)` to navigate — handles cleanup (chat teardown, calendar reset, etc.) ### Two Roles `session.role` = `'parent'` or `'specialist'`. All render functions branch on this. ### Session Object ```javascript session = { id, // auth.uid() role, // 'parent' or 'specialist' — routing only, never display householdId, // UUID — use for household queries, NOT session.id email, displayName, isPrimary, // true = primary parent, false = co-parent profession, // specialist specialty (e.g. "Speech-Language Therapist") — display this, not role } ``` ### UI Primitives `openModal(title, buildFn, maxW)`, `openConfirm(title, msg, danger, onOk)`, `el(tag, attrs, kids)`, `mkBtn(label, cls, onClick)`, `toast(msg, type, ms)` --- ## Module System (esbuild) New features are built as modules in `src/features/` — NOT in index.html. ``` src/ app.js — entry point features/ reports/ index.js — main module: hub, templates, editor, form, preview, routing templates.js — section library (50+ sections, 5 specialties), template CRUD form-builder.js — dynamic form renderer ai-generator.js — client fetch wrappers for /api/report-ai pdf-util.js — jsPDF PDF generation with branding, markdown, Hebrew font styles.js — CSS injected into document ``` ### Bridge Pattern - index.html exposes globals via `window.HUD` (S, re, T, _supa, el, mkBtn, toast, session, DB, etc.) - Modules register renderers on `window.HUD_*` (e.g. `window.HUD_REPORTS`) - index.html has minimal glue: nav items + routing in `renderContent()` + `TAB_NAMES` ### Build ```bash npm run build # esbuild bundles → public/app.bundle.js ``` Vercel runs `npm run build` on deploy.
